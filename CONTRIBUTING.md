# Contributing

Thank you for your interest in contributing to Hugo Bibliography
We welcome suggestions, ideas, and code contributions from anyone interested in improving this project.

## How to Contribute

### 1. Suggest Improvements
If you have an idea, feature request, or bug report, please open an [issue](../../issues) and describe it clearly.  
We encourage open discussion and collaboration around new ideas.

### 2. Make Code Contributions
If you’d like to make direct changes:
1. **Fork** the repository to your own GitHub account.  
2. **Create a branch** for your feature or fix (`git checkout -b my-feature`).  
3. **Make your changes** and **commit** them with clear messages.  
4. **Open a pull request** to the main repository and describe what your PR does.

Your pull request will be reviewed, and once approved, it will be merged into the project.

### 3. Get More Involved
If you’d like to take a more active role in the project:
- **Join the [DHTech Slack](https://join.slack.com/t/dh-tech-group/shared_invite/zt-3f2bkgb91-Za~wcsSLBC5bvHtB2oJpsg)**, a community for digital humanities technologists.  
- In the DHTech Slack workspace, join the **#hugo-bibliography** channel.  
- You’re welcome to **attend our monthly meetings** on the first Friday of every month to discuss ideas, updates, and upcoming goals.

## Questions?
Feel free to reach out via issues or discussions if you’re unsure where to start or want guidance on your first contribution.
